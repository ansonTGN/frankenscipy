perf: fsci-stats ridge_regression — full XtX build O(n·p²) -> symmetric upper-triangle O(n·p²/2)
================================================================================================

Target : crates/fsci-stats/src/lib.rs :: ridge_regression (the (XtX + alpha*I) / Xty build).
Lever  : ridge built the WHOLE XtX with `for k in 0..p1` — computing both xtx[j][k] and xtx[k][j]
         independently, i.e. every symmetric pair twice. XtX is symmetric, so accumulate only the
         upper triangle (`for k in j..p1`, halving the O(n·p²) work) and mirror once afterwards;
         also reuse one row buffer instead of allocating a Vec per sample. Same family as the
         multiple_regression hoist, but here it is a genuine FLOP halving (the old code did the
         full matrix), not just a mirror move.

Isomorphism proof (byte-identical XtX/Xty -> byte-identical beta)
----------------------------------------------------------------
- The old lower entry was xtx[k][j] += row[k]*row[j] accumulated over samples i=0..n; the new code
  produces xtx[k][j] by mirroring xtx[j][k] = Σ_i row[j]*row[k]. IEEE-754 multiplication is
  COMMUTATIVE (row[j]*row[k] == row[k]*row[j] bit-for-bit), summed over the same samples in the
  same order, so the mirrored value equals the old directly-computed lower entry exactly. The
  upper triangle and Xty are unchanged; the reused buffer yields identical row[j]; the alpha*I
  regularization step is untouched. So XtX, Xty, and hence beta are bit-for-bit identical.
- Empirically: 0 mismatches / 2000 random problems (n in [2,81], p in [1,12], random alpha); the
  full XtX and Xty compared via .to_bits() between a verbatim original build and the new build.
