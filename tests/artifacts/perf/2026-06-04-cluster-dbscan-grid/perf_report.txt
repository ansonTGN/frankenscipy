perf: fsci-cluster dbscan — naive O(n^2) all-pairs region queries -> uniform spatial grid
==========================================================================================

Target : crates/fsci-cluster/src/lib.rs :: dbscan (neighbour query).
Lever  : DBSCAN called neighbors(idx) = {j : sq_dist(p_idx, p_j) <= eps^2} by scanning all n
         points, once per visited point -> O(n^2) distance evaluations. Bucket points into a
         uniform grid of cell side `eps`. A neighbour has full distance <= eps, so each
         coordinate differs by <= eps and the two points fall in cells differing by <= 1 per
         axis -> only the 3^d cells around a point can contain a neighbour. For bounded density
         this is ~O(n). New: grid HashMap<cell, Vec<idx>> + 3^d-cell candidate gather.

Scope / gating
--------------
Grid used only when d <= 6 && n >= 256 (3^d stays small and the bucket build amortizes); higher
dimensions / tiny inputs keep the linear scan. Both paths are exact, so dispatch never changes
results.

Isomorphism proof (identical labels / cores / cluster count)
------------------------------------------------------------
- Completeness: a true neighbour (sq_dist <= eps^2) has |Δ_k| <= eps for every axis, so its cell
  differs by at most 1 per axis from the query's cell -> it is in one of the enumerated 3^d cells.
  The grid therefore yields a SUPERSET of candidates.
- Exactness: each candidate is filtered with the SAME `sq_dist_within(pi, pj, eps2) <= eps2` test
  the linear scan used (bit-identical membership), and the result is sorted ascending -> the same
  set in the same 0..n index order the `(0..n).filter` produced. Identical neighbour lists ->
  identical BFS expansion -> identical labels, core_sample_indices, and n_clusters.
- Empirically: 0 mismatches / 1600 dbscan runs across 400 datasets (blobs + uniform noise),
  d in [2,5], n in [256, 955], eps/min_samples in {(0.5,3),(1.0,4),(1.5,5),(2.5,8)}. Labels,
  core indices, and cluster count all compared exactly vs a verbatim O(n^2) copy.
