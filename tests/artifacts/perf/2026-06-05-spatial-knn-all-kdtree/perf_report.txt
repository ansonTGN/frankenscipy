perf: fsci-spatial k_nearest_neighbors — O(n^2) all-pairs -> KD-tree, O(n log n + n k)
======================================================================================

Target : crates/fsci-spatial/src/lib.rs :: k_nearest_neighbors (all-points k-NN).
Lever  : The original computed all m = n-1 distances for every query (then select_nth + sorted
         the k-prefix) -> O(n^2) total. Build a KD-tree once and descend it per query with a
         bounded composite-keyed set: O(log n + k) average per point, O(n log n + n k) total.
         Extends last commit's 1-NN KD-tree lever (02cdbd6c) to general k via `knn_search_composite`
         (a sorted, k-capped `best` set keyed on (squared distance, index)). Falls back to the
         brute force for inputs the tree can't index (ragged rows / non-finite) and for k == 0.

Isomorphism proof (byte-identical: indices AND distance bits)
-------------------------------------------------------------
- Ordering: `best` is kept sorted by the SAME composite key the brute force sorts by
  (`d.1.total_cmp(&d.0... )` i.e. squared-distance total_cmp then ascending index; squared and
  Euclidean order agree, and indices are unique so the key is a strict total order).
- Distances: each kept point's `dist_sq.sqrt()` is the identical `euclidean(a,b)` bits (same
  map/sum/sqrt; the tree stores points verbatim keyed by original index).
- Completeness: `knn_search_composite` prunes a far subtree only when `diff*diff` exceeds the
  current k-th squared distance; since that k-th value never drops below the true k-th smallest,
  every point that could enter the k smallest — including equal-distance points with a smaller
  index — is visited, so the bounded set converges to exactly the brute force's k.
- Empirically: 0 mismatches / 648 cases across n in {1,2,5,17,64,200} x dim in {1,2,3} x
  k in {1,3,7,50} x tie densities grid in {0(continuous),4,9} x 3 seeds, comparing library index
  lists (==) and distance bits (to_bits) to a VERBATIM copy of the original brute force.
- In-tree test `k_nearest_neighbors_kdtree_matches_brute_force_bitwise` re-checks across k and
  tie-heavy grids (incl. k > n). Existing k-NN tests pass (7 nearest/knn lib tests green).
