perf: fsci-spatial directed_hausdorff — naive O(N·M) -> Taha-Hanbury early-break
=================================================================================

Target : crates/fsci-spatial/src/lib.rs :: directed_hausdorff (also powers hausdorff_distance).
Lever  : Directed Hausdorff = max_{a in xa} min_{b in xb} d(a,b). Once an inner squared distance
         drops below the running max `cmax`, that a's minimum is already < cmax and cannot raise
         the max, so abandon its scan (Taha & Hanbury 2015). Iterate both point sets in a fixed
         deterministic Fisher-Yates order so the early break is effective even on
         adversarially-ordered input. New helper hausdorff_scan_order.

Isomorphism proof (byte-identical scalar)
-----------------------------------------
- The function returns only the scalar distance (no index pair), so there is no tie-break to
  preserve. The result is the squared distance of the achieving (a*, b*) pair, computed by the
  same `sqeuclidean`; min/max are value-only (order-independent). Pruning pairs that cannot change
  the result therefore leaves the final f64 bit-for-bit unchanged.
  Proof that a* is never wrongly pruned: when a* is processed, cmax <= V (global max); every
  d(a*, b) >= V >= cmax, and the break is strict `< cmax`, so a* is fully scanned and cmin = V.
- The scan order (Fisher-Yates from a fixed seed) only changes iteration order, not the set, so
  both the per-a minimum value and the overall maximum value are unchanged.
- Empirically: 0 mismatches / 1500 results across 500 trials x {a->b, b->a, symmetric}, mixed
  coarse integer grids (heavy ties / coincident points) and fine coordinates, n,m in [1,60],
  dim in [1,4]. Every result compared via .to_bits() vs a verbatim O(N·M) copy.
