perf: fsci-ndimage chessboard distance transform — brute-force O(F·B) -> chamfer O(N·3^ndim)
=============================================================================================

Target : crates/fsci-ndimage/src/lib.rs :: distance_transform_by_metric (Chessboard metric),
         shared by distance_transform_cdt(chessboard) AND distance_transform_bf(chessboard).
Lever  : Chessboard (Chebyshev / L∞ = max_axis |Δ|) is not additively separable, but every
         neighbour in the full 3^ndim Moore neighbourhood has weight 1, which IS the chessboard
         metric. Replace the O(foreground·background) scan with a two-pass full-neighbourhood
         chamfer (Rosenfeld-Pfaltz): forward raster sweep relaxing from already-visited
         neighbours, then a backward sweep. New helper chessboard_distance_transform.
         This completes the grid-metric family alongside cityblock (commit 52b2e74b).

Scope / gating
--------------
Fast path only when metric == Chessboard && background nonempty. The no-background sentinel
keeps the brute force. Euclidean (Felzenszwalb) and taxicab (separable) keep their own paths.

Isomorphism proof (byte-identical f64)
--------------------------------------
- A two-pass sequential chamfer with the full unit-weight Moore neighbourhood computes the
  exact L∞ distance for binary images (classic Rosenfeld-Pfaltz result); the chessboard metric
  IS that chamfer, so the result equals min over background of max_axis |Δ|.
- All values are exact integers, so the result is byte-identical to the brute-force max-then-min.
- Empirically: 0 mismatches / 7464 cells across shapes {24},{10,10},{16,24},{5,7,9},{1,19},
  {4,4,4},{3,3,3} × fg densities {20,40,60,80}%, comparing BOTH distance_transform_cdt and
  distance_transform_bf vs a verbatim L∞ brute force via .to_bits().
